                                            Системные Namespaces

  Kubernetes включает несколько предопределённых системных Namespace, 
которые используются для организации ресурсов, связанных с системными компонентами Kubernetes.

* default
    Если при создании объекта не указан Namespace, он по умолчанию создаётся в Namespace default. 
Это пространство имён удобно для начала работы, например для запуска тестового Pod. 
Но для более структурированной организации рекомендуется создавать собственные Namespaces.

* kube-system
    В этом Namespace размещаются основные системные компоненты Kubernetes, 
такие как kube-dns, kube-proxy и другие системные Pods. 
Пользовательские приложения не следует развёртывать в этом Namespace в целях избежания конфликтов.

* kube-public
    Этот Namespace доступен для чтения всем пользователям, включая неаутентифицированных. 
Он часто используется для ресурсов, которые должны быть общедоступными.

* kube-node-lease
    Используется для хранения объектов Lease, 
которые помогают отслеживать доступность nod в кластере и управлять их статусом.

* cluster.local    - доменное имя по-умолчанию для кластера кубернетис

* dev              - простанство имён
* db-service       - имя службы

- kubectl config view --minify | grep namespace            - узнать в каком пространстве имён находишься сейчас
- kubectl get events -owide                                - получить все события в пространстве имён
- kubectl get pods --namespace=kube-system                 - получить количество подов в пространстве имён kube-system
- kubectl get pods --all-namespaces                        - получить количество подов во всех пространств имён
  kubectl get pods -A
- kubectl get all -A --show-labels                         - получить все метки в кластере Kubernetes

- kubectl create -f pod-definition.yaml --namespace=dev    - создать под в простраснтве имён = dev
                                                             (или поместить --namespace=dev в файле в секции metadata: внизу kind)  
- kubectl create namespace namespace_name                  - создать пространтсво имён

- kubectl config set-context $(kubectl config current-context) --namespace=dev  - переключиться в пространство имён - dev
- kubectl config set-context --current --namespace=dev     - задать Namespace "dev" по умолчанию
