                                          Pull Request

  Реализовав в отдельной ветке всё, что требовалось, возникает необходимость загрузить изменения на сервер.
Напомним, что для публикации новой ветки нужно использовать команду git push с ключом -u и двумя аргументами:

-  первый аргумент — имя сервера, с которого скопирован репозиторий origin (англ. «источник»);
-  второй аргумент — имя ветки, например, develop (англ. «разрабатывать»);
-  ключ -u — связывает локальную ветку с веткой удалённого репозитория.

Итак, публикуется новая ветка:

git push -u origin develop

Если всё прошло хорошо, вы увидите новую ветку в своём репозитории на GitHub.

----------------------------------------------------------------------------------------------------------------------------------

                                        Делаем pull request

  Итак, новая ветка запушена, все тесты пройдены. Пора делать pull request!
Для этого перейдите в репозиторий. Там вы увидите кнопку Compare & pull request:

Напишите название и описание pull request-а:

Потом нажмите зелёную кнопку Create pull request. Можно скопировать ссылку на получившийся pull request и отправить коллегам.

После окончания ревью во вкладке Pull requests можно будет увидеть комментарии и обсудить изменения:

После этого тимлидер объединит ваш код с оригинальным кодом ветки main — нажмёт кнопку Merge pull request или попросит вас сделать это.
