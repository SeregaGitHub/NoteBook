        
        Imperative  - подход (прямое указание)
                                        kubectl

- kubectl run --image=nginx nginx                        - создание подов
- kubectl create deployment --image=nginx nginx          - создание деплойментов
- kubectl expose deployment nginx --port 80              - создание службы(сервиса) для деплоймента (nginx)

- kubectl edit deployment nginx                          - редактирование ресурса
- kubectl scale deployment nginx --replicas=5            - изменения для масштабирования
- kubectl set image deployment nginx nginx=nginx:1.18    - обновление образа в деплойменте удаление сервиса

- kubectl create -f nginx.yaml                           - создание ресурса из yaml файла        
- kubectl replace -f nginx.yaml                          - изменение ресурса из yaml файла
- kubectl replace --force -f nginx.yaml                  - изменение ресурса из yaml файла (путём удаления и создания ресурса заново)
- kubectl delete -f nginx.yaml                           - удаление ресурса из yaml файла


- kubectl delete svc service_name                        - удаление сервиса
- kubectl exec pod_name -- cat /log/app.log              - просмотр файла app.log внутри контейнера
