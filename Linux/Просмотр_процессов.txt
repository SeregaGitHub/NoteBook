
                                                                Команда ps

    Посмотреть список запущенных процессов можно командой ps (от англ. process status — «состояние процесса»). 
В зависимости от версии и сборки ОС, на экран могут быть выведены разные столбцы. 

При запуске команды внутри нашего тестового Docker-контейнера столбцы будут следующие:

    PID — Process ID, идентификатор процесса;
    USER — пользователь, который запустил данный процесс;
    TIME — время, затраченное на обработку этого процесса;
    COMMAND — команда, запустившая данный процесс.

    Если процессов много, их можно отфильтровать с помощью команды grep. 
Через символ | указывается команда grep и необходимая информация для поиска. 
Например, если запущен редактор nano, ps | grep nano отобразит всего два процесса: 
команды с запуском редактора и сам процесс ps, где использовалась команда.


                                                                Команда top

    Чуть больше информации несёт команда top, которая отображает данные по процессам в реальном времени. 
Например, можно увидеть, насколько сильно загружен процессор (столбец %CPU) для каждого процесса.

    Выйти из режима просмотра можно или нажав клавишу q, или остановив выполнение сочетанием Ctrl+C.
Как и в ps, в top можно использовать grep. Команда top | grep nano отобразит только процесс с упоминанием nano.


                                                            Как остановить процесс

    Остановить нежелательный процесс, например, если он завис и больше не отвечает, можно с помощью PID — уникального номера процесса. 
Понадобится команда kill [ключ] [PID] (англ. kill — «убить»), где ключ указывает на тип сигнала.
    Если ключ не указывать (к примеру, для PID = 77 команда будет выглядеть как kill 77), 
то по умолчанию передаётся сигнал SIGTERM (англ. signal terminated — «сигнал завершения»). 
Его можно сравнить с нажатием на кнопку «Закрыть» в Windows.
    Однако в некоторых случаях сигнал SIGTERM может быть проигнорирован программой. Поэтому, особенно если речь идёт о зависших процессах, 
часто используют сигнал SIGKILL — принудительное и немедленное завершение процесса. Эта команда аналогична «Завершить сейчас» в Диспетчере задач Windows.
    Каждому сигналу соответствует числовой ключ, который указывается после команды kill. 
Для SIGKILL это ключ -9, поэтому команда в итоге будет выглядеть как kill -9 [PID]. 
Ознакомиться с сигналами подробнее можно в документации:
https://pubs.opengroup.org/onlinepubs/9699919799/utilities/kill.html#tag_20_64
