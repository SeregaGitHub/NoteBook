                        Write-through: запрос клиента пишет в кэш, а тот в БД



                    Read-through: запрос клиента читает кэш, если там нет - то в БД



                    Cache-aside: сервер пишет и в кэш и в БД (Для Redis самый частый)

Чтение:     проверка данных в кэше -> если есть - отдать клиенту, если нет - отдать клиенту + сохранить в кэш
Обновление: (викшен) удалить данных из кэша -> поместить в кэш обновлённые
