================================================================================================================

    Добавьте в корень файл с названием Dockerfile и без расширения.
Dockerfile — текстовый документ с короткими и понятными инструкциями для сборки образов.
Они служат схемой, по которой Docker будет собирать образ для контейнеров.
Каждая инструкций пишется заглавными буквами, а их выполнение происходит последовательно — одна за другой.

    Откройте Dockerfile, добавьте в него инструкции для сборки образов и сохраните изменения.

FROM amazoncorretto:11-alpine-jdk
COPY target/*.jar app.jar
ENTRYPOINT ["java","-jar","/app.jar"]

    Разберём подробнее, что это за команды.

- FROM — Dockerfile почти всегда начинается с этой инструкции. Она определяет базовый образ, на основе которого создаётся локальный образ.

FROM <image>:<tag>
# <image> — имя базового образа — например, amazoncorretto
# <tag> — необязательный параметр, указывает конкретную версию образа — 11-alpine-jdk


    Если не указать версию образа, будет установлена последняя версия с тегом latest (англ. «последний»).
Но хорошей практикой всё же считается явное указание версии.

    В Dockerfile для вашего проекта инструкция записана так.

FROM amazoncorretto:11-alpine-jdk

    В основе большинства образов лежит дистрибутив операционной системы.
В данном случае это дистрибутив Linux с предустановленным интерпретатором Java 11 от Amazon Corretto.
Версия alpine говорит, что дистрибутив построен на легковесной системе Linux.
Ознакомиться с другими тегами можно на странице образа в DockerHub.

- COPY — копирует файлы и директории из указанной локальной директории в директорию контейнера.

COPY <файл/директория> <путь-внутри-контейнера>
# альтернативный вариант записи:
COPY ["<файл/директория>", "<путь-внутри-контейнера>"]

    Если целевой директории для копирования нет, то инструкция COPY создаёт её.
В вашем Dockerfile все файлы, подходящие по маске target/*.jar (это только что собранный .jar),
копируются в новый архив app.jar, который будет создан внутри образа.

COPY target/*.jar app.jar

- ENTRYPOINT — консольная команда, которая выполнится внутри контейнера после его запуска.

ENTRYPOINT command param1 param2
# или
ENTRYPOINT ["executable", "param1", "param2"]

В данном случае после запуска контейнера выполнится команда java -jar /app.jar, которая запустит Java-приложение.

ENTRYPOINT ["java","-jar","/app.jar"]

    Инструкции Dockerfile описаны. Узнать об инструкциях больше поможет документация:
https://docs.docker.com/engine/reference/builder/#from
