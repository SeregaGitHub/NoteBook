noda - минимальная единица в кластере
    
    main noda - 

* kube API Server    - оркестрация всех операций
* ETCD cluster       - хранение информации о кластере (ключ - значение)
* scheduler          - планирование приложений на узлы (pods on nodes)
* controller manager - управление нодой, репликация, нагрузки и т.п. (отслеживание состояний объектов)

-------------------------------------------------------------------

    work noda -

* kubelet            - слушает инструкции от kube API Server и управляет нодой (создаёт поду на ноде)
* kube proxy         - обеспечение связи между службами в кластере
* container runtime
-- если ставить не в ручную - это всё так же будет и в main noda !!!
