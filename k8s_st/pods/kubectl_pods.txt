                                        kubectl

- kubectl get pods                                 - сколько под в кластере
                    -o wide                        - дополнительная информация о подах
                    --selector app=webapp          - выбрать поды с меткой - app, значение которых - webapp
                    --no-headers | wc -l           - узнать количество подов
                    --show-labels                  - узнать метки
- kubectl get pods --all-namespaces -o wide        - найти под с именем current_pod_name
  | grep current_pod_name
- kubectl describe pod pod_name                    - подробное описание пода
- kubectl run pod_name --image=image_name          - создание пода
                       --labels="app=th,tier=db"     - добавление меток
- kubectl get pod/pod_name -o jsonpath="{..image}" - узнать из какого образа развёрнут под
- kubectl describe pods                            - узнать состояние контейнеров
- kubectl describe pod --help                      - справка по команде "kubectl describe"
- kubectl delete pod pod_name                      - удалить под
                     --all                         - удалить все поды

                        Создание пода с redis из файла манифеста

- создать файл манифеста
  * новый pod с названием redis
  * из образа redis123
  kubectl run redis --image=redis --dry-run=client -o yaml > redis-definition.yaml
  kubectl run redis --image=redis --restart=Never --dry-run=client -o yaml --command -- sleep 1000 > redis-definition.yaml
                                  (конфигурация контейнера)                (команда для контейнера)

- создать ресурс из файла-манифеста
  kubectl create -f redis-definition.yaml  

- проверить 
  kubectl get pods

- проверить с доп инфой
  kubectl get pods -o wide

При существующем поде, создать для него файл манифеста  
kubectl get po pod_name -o yaml > pod_name.yaml
