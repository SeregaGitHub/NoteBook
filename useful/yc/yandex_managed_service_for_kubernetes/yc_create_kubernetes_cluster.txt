  В консоли Управления Yandex Cloud выберете каталог, в котором будете создавать ресурсы, 
нажмите Создать ресурс и выберете Кластер Kubernetes

                                    Общие настройки

    Имя
Наименование кластера, уникальное в рамках каталога.

    Сервисный аккаунт для ресурсов
Будет использоваться для создания ресурсов в облаке. Если выбрать «Автоматически»,  будет создан сервисный аккаунт с требуемой ролью. 

    Сервисный аккаунт для узлов
Будет использоваться рабочими узлами для доступа к реестру Docker-образов. 
Если выбрать «Автоматически», будет создан сервисный аккаунт с требуемой ролью. 

    Ключ шифрования
Будет использоваться для шифрования секретов.

    Релизный канал
Это канал получения обновлений кластера. Их существует три:

- RAPID
Содержит последние версии Kubernetes. Частые обновления с новой функциональностью и улучшениями. 
Нельзя отключить автообновления. Подходит для DEV-, TEST- и STAGE-окружений.

- REGULAR 
Содержит различные версии Kubernetes. Обновления поступают постепенно, 
через некоторое время после появления на канале RAPID. 
Можно отключить автообновления. Подходит для DEV-, TEST-, STAGE-окружений.

- STABLE 
Содержит стабильные версии Kubernetes. Выпускаются обновления, направленные на улучшение безопасности и исправление ошибок. 
Можно отключить автообновления. Рекомендуется для PROD-окружений.

❗ Изменить канал после создания кластера нельзя. Возможно только пересоздать кластер и указать новый релизный канал.


    Добавьте группу рабочих узлов, перейдя на вкладку Управление узлами и нажав Создать группу узлов.


                                                Как подключиться к кластеру

Если у вас ещё не установлен интерфейс командной строки Yandex Cloud, установите и проинициализируйте его с помощью инструкции.

Установите по инструкции утилиту kubectl.

В консоли управления Yandex Cloud на странице вашего кластера нажмите "Подключиться" в правом верхнем углу

Выполните команду:

* yc managed-kubernetes cluster get-credentials --id <ID_кластера> --external 

В результате будут добавлены учётные данные для подключения к кластеру в файле $HOME/.kube/config.
Теперь можно использовать утилиту kubectl для работы с кластером. 
Запросите информацию о кластере:

* kubectl cluster-info 

Если конфигурация kubectl настроена верно, команда вернёт информацию о кластере

Просмотрите список рабочих узлов вашего кластера:

* kubectl get nodes
