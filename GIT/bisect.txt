  С помощью команды git bisect можно переключаться между коммитами до обнаружения коммита с ошибкой.

Алгоритм git bisect работает по принципу бинарного (двоичного) поиска.

                                            Применение git bisect:

Последовательность работы с git bisect в большинстве случаев будет выглядеть так:

Открываем сеанс поиска коммита git bisect start

- git bisect start 

В результате выполнения этой операции Git уведомит об ожидании плохих или хороших коммитов:

status: waiting for both good and bad commits 

Помечаем текущий коммит как «плохой»

- git bisect bad 

Указываем хеш хорошего коммита (когда мы уверены, что в коммите всё работает): 

- git bisect good <хеш_хорошего_коммита> 

Происходит переключение на промежуточный коммит (посередине между хорошим и плохим).
Проверяем работоспособность и, если ошибка сохраняется, то выполняем git bisect bad. А если ошибка исчезла — git bisect good. 
Повторяем этот шаг до тех пор, пока Git не уведомит о нахождении первого плохого коммита:

------------------------------------------------------------------------------------------------------------------------------
<commit_hash> is the first bad commit
commit <commit_hash>
Author: ДартВейдер <kk@>
Date:   Wed Nov 13 21:05:02 2024 +0300
    Сообщение коммита
 # Далее идёт информация о внесённых изменениях 
------------------------------------------------------------------------------------------------------------------------------

Если возникнет необходимость прервать поиск первого плохого коммита, то нужно выполнить  

- git bisect reset

чтобы в следующий раз начать сначала.
