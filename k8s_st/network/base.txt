                                    Base Network

-----------------------------------------------------------------------------------------------------------------

- ip link                                      - интерфейсы для хоста
- ip addr                                      - просмотр ip адресов 
- ip addr add 192.168.1.10/24 dev eth0         - назначить ip адрес для хоста 
- ping 192.168.1.10                            - проверка доступности хоста

                        ( действительны только до перезапуска !!! )

-----------------------------------------------------------------------------------------------------------------

- route                                        - таблица конфигурации маршрутизации

- ip route add default via 192.168.2.1         - установка шлюза 192.168.2.1 по-умолчанию
  ip route add 0.0.0.0 via 192.168.2.1

=================================================================================================================

                            Соединение необходимо настраивать с двух сторон

- ip route add 192.168.2.0/24 via 192.168.1.6  - подключение к сети 192.168.2.0/24 через шлюз 192.168.1.6
                                                 (чтобы достучаться до хостов 192.168.2._ через хост 192.168.1.6)

             some_host
    192.168.1.6 -:- 192.168.2.6                - соединительный хост

- ip route add 192.168.1.0/24 via 192.168.2.6  - подключение к сети 192.168.1.0/24 через шлюз 192.168.2.1
                                                 (чтобы достучаться до хостов 192.168.1._ через хост 192.168.2.6)

-----------------------------------------------------------------------------------------------------------------

* По-умолчанию, в linux, не разрешено передавать пакеты из одной сети в другую, можно изменить это

                                        some_host

- cat /proc/sys/net/ipv4/ip_forward            -  0 - не разрешено
- echo 1 > /proc/sys/net/ipv4/ip_forward       -  1 - разрешено

- /etc/sysctl.conf    net.ipv4.ip_forward = 1  - чтобы действовало постоянно !!!
=================================================================================================================

                            Из локальной сети 192.168.1._ в www

- iptables -t nat -A POSTROUTING -s 192.168.1.0/24 -j MASQUERADE    - адрес отправителя будет заменён на адрес роутера
