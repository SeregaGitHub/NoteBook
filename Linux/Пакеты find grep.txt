			Распаковка архивов:

unzip архив.zip          -  распаковать содержимое архива.zip, исходный архив не удалится
gunzip архив.gz          -  распаковать содержимое архива.gz, исходный архив будет удалён
tar -xvf архив.tar       -  распаковать архив.tar
tar -xzvf архив.tar.gz   -  распаковать архив.tar.gz (с использованием gunzip)
unxz архив.tar.xz

			Создание архивов:

zip архив.zip файл1 файл2 ...           -  запаковать перечисленные файлы и/или папки в архив.zip
gzip файл                               -  запаковать файл в файл.gz, исходный файл удалить
                                           (напрямую нельзя запаковать gzip несколько файлов)
                                           (если это нужно то: => вниз...)
tar -cvf архив.tar файл1 файл2 ...  (1) -  запаковать перечисленные файлы
                                           и/или папки в архив.tar (без сжатия)
(tar tf архив.tar - просмотр содержимого архива)       (потом => вниз...)

gzip архив.tar                      (2) -  запаковать архив.tar в архив.tar.gz,
                                           исходный архив.tar удалить
                                               (ИЛИ разом => вниз...)
tar -zcvf архив.tar.gz файл1 файл2 ...  -  запаковать перечисленные файлы и/или папки
                                           в архив.tar.gz (c сжатием
                                           при помощи gzip
xz архив.tar

			Еще один архиватор:
bzip2 файл
bunzip2 файл.bz2
tar -cjvf архив.tar.bz2 файл1 файл2 ...
tar -xjvf архив.tar.bz2

			Сжатие/распаковка без удаления:
gzip -c файл > файл.gz
gunzip -c файл.gz > файл

bunzip2 -c файл.bz2 > файл

find . -type f -name "test*"           - поиск файлов (-type f  -  файл)
               -iname                  - поиск файлов без учёта регистра
                     "test?.txt"       - поиск файлов txt, начинающихся с test + 1 символ
       -type d                         - поиск папок
                                                   (-name    -  по имени)
                                                   ("test*"  -  начинаются с test)
                                                   ("*.txt"  -  с расширением .txt)
find . -type f -not -name "*.txt"      - поиск файлов НЕ с расширением .txt

find . -type f -perm 0664              - поиск по правам доступа
find . -size +1M                       - поиск всех более 1 мегабайта
find . -size -10k                      - поиск всех менее 10 килобайт
find . -type f -mtime 30               - поиск файлов модифицированных 30 дней назад
find . -type f -atime 30               - поиск файлов открытых 30 дней назад
find . -maxdepth 1 -type d -name ".*"  - поиск скрытых папок (".*")

grep "am" fileName.txt               -  может найти слово

grep "Hell" fileName.txt             -  может найти часть слово

grep -n "am" fileName.txt            - -n укажет строку на которой слово

grep -i "Am" fileName.txt            - -i находит слово в независимости от регистра
I am glad to see you !!!

grep -c "Am" fileName.txt            - -c подсчитать количество "Am"

grep "am" fileName.txt > Answer.txt  -  найти слово am и записать
                                        все строки в файл Answer.txt

grep -r "Am" myDir                   - -r найти "Am" во всех файлах в папке myDir


  В некоторых случаях важно получить не только строку, но и окружающую её информацию.
Например, посмотреть в логах, что происходило с приложением до ошибки. Вывести соседние строки помогут ключи -B, -A и -C.

    -B (--before-context) — количество отображаемых строк до искомой;
    -A (--after-context) — количество отображаемых строк после искомой;
    -C, (--context) — количество отображаемых строк до и после искомой одновременно.

  Ключи -B, -A и -C обязательно использовать с указанием нужного количества дополнительных строк.
Например, ключ -C со значением 1 выведет искомую строку и строки на одну выше и ниже искомой. Введём grep -C 1 Хьюстон logs.txt.
