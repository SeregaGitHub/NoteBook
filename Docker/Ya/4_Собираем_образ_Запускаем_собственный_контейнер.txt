
                                                      Собираем образ

  Откройте терминал. Убедитесь, что вы находитесь в той же директории, где сохранён Dockerfile, а сам Docker запущен.
Запустите сборку образа.

docker build -t dock_image . 

Рассмотрим части этой команды:

-      build — команда сборки образа по инструкциям из Dockerfile.
-      -t <имя образа> — ключ, который позволяет задать имя образу (в нашем примере это dock_image).
-      .  — точка в конце команды, путь до Dockerfile, на основе которого производится сборка.
    Обычно Dockerfile располагают непосредственно в директории для сборки, из которой и запускается docker build, — поэтому указывается только точка.


    Удалить лишние образы можно с помощью следующей команды:

- docker image rm image_id


                                                  Запускаем контейнер

    Чтобы запустить контейнер, используйте следующую команду в терминале.

docker run --name dock_container -p 8080:8080 dock_image

Разберём элементы команды:

-    run — команда запуска нового контейнера.
-    --name <имя контейнера> — ключ, который позволяет задать имя контейнеру. Если параметр не указан, Docker сгенерирует его самостоятельно.
-    -p 8080:8080 — публичный порт контейнера. Левая часть — его внешний порт, правая — порт, на который будет перенаправлен запрос внутри приложения.
-    dock_image — образ, из которого будет запущен контейнер.
