                                            Деплоймент

- kubectl create -f deployment-definition.yaml                    - создание файла деплоймента
                                                                    ( автоматически создаст реплика сет )
- kubectl create deployment deployment_name --image=image_name    - создание деплоймента из образа image_name с именем deployment_name
- kubectl apply -f deployment-definition.yaml                     - обновление деплоймента из файла манифеста
- kubectl edit deployment deployment_name                         - редактирование деплоймента
- kubectl set image deployment/deployment_name nginx=nginx:1.7.1  - обновление образа nginx в деплойменте     
- kubectl scale deployment deployment_name --replicas=2           - обновление количества реплик в деплойменте                                                           
- kubectl get all                                                 - просмотр всех подов, реплика сетов, деплойментов
- kubectl get deployments -o wide                                 - образ использованый для создания pods в Deployment
- kubectl describe deployment deployment_name                     - описание деплоймента
- kubectl rollout status deployment/deployment_name               - получить состояние выкатки
- kubectl rollout history deployment/deployment_name              - получить историю выкатки
- kubectl rollout undo deployment/deployment_name                 - откат обновления приложения
